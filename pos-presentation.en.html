Practice, not theory

Build one
operating loop

01 · CAPTURE

Answer the intake

Give the bot concrete context: role, project, blocker, support needed, and current AI use.

02 · CONTEXT

Create CLAUDE.md

Use /prompt to turn your answers into a short working agreement for Claude Code or Codex.

03 · CHOOSE

Pick one workflow

Start with one repeatable task: weekly plan, research brief, GitHub issue, or product audit.

04 · RUN

Use AI on real work

Do not demo an abstract prompt. Run it on something you actually need this week.

05 · SAVE

Store the result

Put the output into your vault, repo, issue tracker, or calendar so it survives the chat.

06 · IMPROVE

Upgrade the loop

Only after the loop works, turn it into a skill, connector, or lightweight agent.

How it works inside

Skills, MCP
and CLAUDE.MD

CLAUDE.MD / AGENTS.MD

The main AI-agent configuration file - global or project-level. It sets context, rules, and behavior from the first second.

MCP
📅 Calendar ⚡ Linear ✈️ Telegram 📁 Files 💻 Code

Like USB ports for an AI agent. Through MCP, the agent connects to Calendar, Linear, Telegram, and any other tool.

One standard - not custom hacks for every tool
SKILLS

A Markdown file with step-by-step instructions for the agent. Like an SOP for an employee: you do not explain the task from scratch every time.

Reusable system prompts with files - more structured, and activated by triggers
Resources load only when needed
A trigger activates the right skill automatically
One skill - one repeatable practice

Key transition

From tools
to cognitive infrastructure

BEFORE
  • A set of disconnected AI tools
  • Memory disappears between sessions
  • Tasks live separately from the person
  • Tools compete with each other
  • AI is used only when you remember to ask
→
AFTER
  • Personal cognitive infrastructure
  • Memory accumulates and connects
  • The system knows you and your goals
  • Tools form one operating loop
  • AI helps at the exact step where work gets stuck

After the workshop

Start small.
Keep the loop.

Day 1
- create a personal context file with /prompt
Day 2
- choose one workflow where AI can remove friction
Day 3
- run it on a real task and save the result
Day 4-6
- repeat, then turn the pattern into a reusable prompt or skill
Day 7
- decide what to connect next: notes, tasks, calendar, code, or search
The goal is not to learn every tool.
The goal is to build one useful loop you will keep using.
