The simple POS model

The simple POS loop

Context -> Procedure -> Tool -> Output -> Feedback

Context

What AI needs to know about you, your work, and your standards.

Procedure

The steps AI should follow, not just a clever prompt.

Tool

Where the work happens: chat, docs, code, notes, calendar, bot.

Output

A useful artifact: plan, summary, deck, checklist, decision, code.

Feedback

What you save so the next run gets better.

This is enough for the first version. Agents can come later.

Step 1

Write a small context file

01
Who I amRole, background, current situation.
02
What I am buildingProjects, goals, current priorities.
03
How I workStyle, constraints, rhythm, decision habits.
04
What good looks likeExamples, tone, quality bar, formats you prefer.
Prompt to use today
Interview me one question at a time and create a 2-page personal context file for working with AI. Focus on: - what I am building - my goals this month - my constraints - my preferred style - examples of good output At the end, produce a clean markdown file.
Step 2

Turn one repeated task into a workflow

Trigger
When do I run it?Example: every Sunday, after a meeting, before publishing.
Input
What do I give AI?Notes, transcript, goal, rough idea, task list, screenshots.
Steps
What should AI do?Clarify, structure, draft, compare, improve, check.
Output
What should come out?A plan, summary, message, deck, checklist, code patch.
Done when
How do I judge it?Clear, usable, specific, saved, ready for next action.

If you cannot define the output, the workflow is not ready yet.

Superpowers lesson

Procedure beats clever prompting

Rule 1
Clarify before doingAsk: what is the goal, what are the constraints, what would success look like?
Rule 2
Plan before buildingAsk AI to show the steps before it creates the final output.
Rule 3
Verify before trustingAsk for checks, assumptions, gaps, and evidence before you use the result.

Superpowers is useful here as a principle: good AI work is a repeatable operating procedure, not a magic prompt.

Workshop exercise

3. Write procedureDefine trigger, input, steps, output, done when.
4. Run onceUse any AI tool. Do not optimize too early.
5. Save feedbackWhat worked, what failed, what should change next time?
